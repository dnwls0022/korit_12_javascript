### 변수 명명 규칙

1. 문자,숫자, \_, $ 를 포함하여 만들수있다.
2. 변수명은 숫자로 시작할수없다.
3. js예약어는 사용할수없다.(console과 같은 변수명으로 쓸 수 없다)
4. 변수명은 대소문자를 구분한다

- ex에서 변수명과 함수명은 camel case 사용/ 클래스명과 생성자명에는 pascal case 권장

- var선언자
- js는 java와는 달리 데이터 타입에 상관없이 변수를 사용할 때 var/let을 사용할수있다. 현재는 오류를 최소화하기위해서 주로 let을 사용하지만 이미 많은 js프로그램이 var로 되어있기때문에 혹시 유지보수팀으로 들어가게될경우 var선언자를 많이 보게된다.

```js
var x = 5;
var y = 6;
var z = x + y;
```

```js
var x = 5;
var y = 6;
var z = x + y;
console.log(z);

var x = 7; //여기서오류 발생해야함 원래는
z = x + y;
console.log(z);
```

오류가발생하지않고 정상실행되는점을봤을때 var선언자를 사용하게되면 변수명이 겹치더라도 오류가나지않기때문에 결과값이다르게 발생할수잇따.

동일 변수명으로 새로운 변수가선언되면 기존코드에서 동일한 이름으로 사용되던 변수에 할당된 값이 사라지므로!! 해당변수를 참조하고있는 프로그램에서 오류가발생할가능성이있다.

그래서 es6이후로 let선언자가생겼고 오늘날 let을 쓸것을 권장한다

const선언자
const는 **Constant(상수)**의 줄임말 
상수 - 변화하지않는 값을가지는 변수
선언시점에 값을 할당한 후에 변수로 재선언 및 재대입 불가

자바의 경우 final키워드는 조심스레사용했어야함
하지만js 의 경우 좀 더 적극적으로 const를 사용한다.

js의 변수는 브라우저에서 값을 바꿀수 있기 때문에 모두가 동일한 결과값을 가지기 힘들다. 하지만 상수로 선언하면 값이 바뀌지않기 때문에 다수의 사용자가 동일한 결과값을 얻을수있다는 점 때문에 const사용을 권함.

주의 함수의 결과값을 const로 집어넣는다는의미가있다.
